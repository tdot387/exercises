## HTML

    Die HTML ist vernünftig strukturiert
    Im Head fehlt der Link zu einem Favicon
    Ein Label für das Input-Feld würde aus Barrierefreiheitsaspekten Sinn ergeben. Placeholder sind für nicht eingeschränkte Personen zwar schön, vernachlässigen allerdings Screenreader und eingeschränkte Personen.

 

## CSS

    Wiederverwendbare Items wie etwa die Farbe bitte in der :root definieren
    Gut strukturiert
    Für größere Projekte lieber Klassen verwenden statt IDs direkt anzusprechen (IDs haben höhere Spezifizität, Klassen können besser wiederverwendet und gewartet werden)

 

## JavaScript

    Arrow-Funktionen verwenden
    Gut, dass der DOM erst vollständig geladen wird, bevor der User damit interagiert
    Du könntest die Funktionen noch kommentieren, damit andere Coder sofort sehen, was innerhalb der Funktionen passiert
